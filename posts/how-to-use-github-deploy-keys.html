Create an SSH Key on Your Server

A GitHub deploy key is, ultimately, an SSH key. So you‚Äôll start by creating an SSH key on your server.

First, connect to your server and open a terminal. Once you‚Äôre in, create an SSH key by executing the following command:

  2. -C USERNAME@EMAIL.com to append your email as a comment at the end of the key. Make sure to replace USERNAME@EMAIL.com with your actual email.

Add the Key to SSH Config

These commands will create an SSH config file, set the correct permissions, and open it using vim. You can also use nano to open the file if you‚Äôre unfamiliar with vim‚Äôs shortcuts.

An SSH config file consists of sections specifying the instructions required for each server you‚Äôll connect to. So, in this case, you‚Äôll provide the instructions needed to connect to your repository. For that, copy this text to the file:

‚ÄåIn the code snippet above, you provide the settings required to connect to GitHub using the SSH key you just created. You specify the following parameters:

  • Host: the name you‚Äôll use in the terminal when referring to this server. Choose a name that‚Äôs easy to remember.
  • HostName: the real hostname that you‚Äôll connect to. In this case, github.com.
  • AddKeysToAgent: specifies if it should add the private key to the ssh-agent.
  • PreferredAuthentications: order in which the client tries authentication methods. In this case, you only use your publickey.
  • IdentityFile: specifies a file from which the key is read. You need to specify the name of the private key you generated earlier. If you used the default name, it should be ~/.ssh/id_ed25519.

First, copy the public key from the SSH key you created earlier. The easiest way of doing that is to run:

Then select and copy the resulting text from the command line. It should look something like this:

Next, you can go back to your server‚Äôs terminal and clone your repository. You can do that by running this command:

This will clone your app‚Äôs repository to your server. Please remember to replace github-YOUR-APP by the Host you specified in the SSH config file. Otherwise, it won‚Äôt work.
